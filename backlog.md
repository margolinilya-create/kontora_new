# Backlog (v3+)

Всё, что сознательно отложено из v1 плана и не вошло в frontend-only v2/v3 сайта. Каждый пункт содержит estimate, зависимости и примерный приоритет.

**Критерий включения в v3:** запуск сайта `new.kontora.su` → 2 недели прогонки → владелец возвращается с реальными бизнес-болями → выбираем что решать первым из этого backlog.

---

## P0 — критично после запуска (≤ 1 месяц)

### 0.1 Реальные фото продукции

- **Проблема:** Сейчас все галереи и product-card'ы используют CSS-плейсхолдеры в брендовых цветах. Визуально ок для MVP, но продажный B2B-сайт без реальных фото кейсов выглядит слабо.
- **Что нужно:** 40–50 фотографий готовых тиражей (по 6–8 на каждую из 7 продуктовых страниц) + 6 образцов плёнок для секции «Материалы».
- **От владельца:** организовать фотосессию в цехе + обработка.
- **Разработка:** 1 день — разложить в `public/gallery/<slug>/`, заменить CSS-тайлы в `Gallery.tsx` на `next/image`, прогнать visual regression.
- **Estimate:** 1 день после получения фото.

### 0.2 Реальная формула калькулятора

- **Проблема:** `lib/pricing/calculate.ts` — placeholder, возвращает `placeholder: true`. UI показывает плашку «Предварительный расчёт», но цифры условные.
- **Что нужно:** коэффициенты от владельца для всех 7 продуктов × 7 материалов × 3 ламинации + min тиражи + надбавки за smalс/надсечку + скидки за тираж.
- **От владельца:** табличка Excel с ценами либо формула в тексте.
- **Разработка:** заменить `BASE_PRICE_PER_CM2`, `MATERIAL_MULTIPLIER`, `LAMINATION_SURCHARGE`, `QTY_DISCOUNT_TIERS` в `lib/pricing/table.ts` + обновить тело `calculate()`. Unit тесты уже готовы и поймают регрессии. Убрать `placeholder: true` из output.
- **Estimate:** 0.5 дня на замену + 0.5 дня на расширение unit-тестов с реальными числами.

### 0.3 FAQ-тексты на продуктовых страницах

- **Проблема:** `faq: []` пусто во всех `lib/content/products/*.ts`. Аккордеон-компонент готов, JSON-LD FAQPage генерируется — но без контента секция не рендерится.
- **Что нужно:** 4–6 вопросов + ответов per product от владельца (или дежурные вопросы из реального саппорта Конторы).
- **Разработка:** правка `lib/content/products/<slug>.ts` через GitHub web editor + обновление `pdf-extract.txt`. Всё через CI, ничего руками не трогать.
- **Estimate:** 0.5 дня после получения текстов.

### 0.4 Реальные telephone/email/telegram в контактах

- **Проблема:** `lib/content/contacts.ts` содержит placeholder телефон `+7 (911) 000-00-00` и `@kontora_su`.
- **Что нужно:** финальные контакты от владельца.
- **Estimate:** 15 минут.

### 0.5 Покупка Druk Wide

- **Проблема:** Display-шрифт сейчас заменён Onest Display (fallback). Брендовая стилистика страдает — заголовки не попадают 1-в-1 в Figma.
- **Что нужно:** владелец покупает лицензию на Druk Wide (~$400, Commercial Type), получает `.woff2` файлы.
- **Разработка:** положить файлы в `public/fonts/DrukWide-*.woff2`, раскомментировать `localFont` блок в `lib/fonts.ts`, удалить заглушку.
- **Estimate:** 30 минут после покупки.

## P1 — средний приоритет (1–3 месяца)

### 1.1 Админ-панель (минимальная)

- **Scope:** приёмка заявок через форму → табличка заявок → смена статуса.
- **Без:** kanban, drag-n-drop, Prisma migrations, роли.
- **Стек:** Supabase (БД + auth + RLS) + `/admin` как protected route.
- **Estimate:** 5–7 дней.
- **Зависимость:** решение владельца — продолжать на Next.js или вынести в отдельный инструмент (например n8n + Google Sheets).

### 1.2 Real-time загрузка макетов

- **Проблема:** Калькулятор сейчас загружает макет в state (data URL), но не передаёт на сервер. Форма через Resend только отправляет метаданные, сам файл теряется.
- **Что нужно:** загрузка в объектное хранилище (Vercel Blob / Cloudflare R2 / Supabase Storage), передача URL в Resend.
- **Estimate:** 2–3 дня.
- **Зависимость:** выбор хранилища.

### 1.3 Checkout + оплата (ЮКасса СБП)

- **Scope:** 3 шага (контакты → доставка → оплата) + интеграция с ЮКасса API + webhook статусов.
- **Estimate:** 7–10 дней.
- **Зависимость:** юрлицо Конторы подключено к ЮКассе.

### 1.4 Интеграция СДЭК / Почты России

- **Scope:** API для расчёта стоимости доставки + создание ТТН + трек-номер.
- **Estimate:** 3–5 дней.
- **Зависимость:** 1.3 (checkout)

### 1.5 CMS для блога

- **Варианты:**
  - **MDX** в репо — бесплатно, владелец пишет через GitHub. 1 день работы.
  - **Sanity Free** — визуальный редактор. 3 дня.
  - **TinaCMS** — inline editing на сайте. 3–4 дня.
- **Рекомендация:** MDX — уже есть дисциплина редактирования через GitHub.
- **Estimate:** 1 день для MDX.

## P2 — полезное (3–6 месяцев)

### 2.1 Pricing rule builder в админке

- **Scope:** визуальный конструктор формул (из v1 плана §6.7) — приоритеты, условия, модификаторы.
- **Когда делать:** когда простой table из `lib/pricing/table.ts` перестаёт быть достаточным (много правил, часто меняется, владелец хочет менять без разработчика).
- **Estimate:** 10–15 дней.

### 2.2 Личный кабинет клиента

- **Scope:** `/account` с историей заказов + трекинг статуса + скачивание документов.
- **Зависимость:** админка (P1.1) + checkout (P1.3).
- **Estimate:** 5–7 дней.

### 2.3 Command Palette (⌘K)

- **Scope:** быстрый поиск по продуктам и действиям. B2B-фича.
- **Зависимость:** реальная аналитика показывает, что B2B-пользователи много кликают по меню — иначе не стоит делать.
- **Estimate:** 2 дня.

### 2.4 A/B тестирование

- **Scope:** framework для вариантов hero/CTA. Обычно через GrowthBook или Vercel Edge Config.
- **Зависимость:** трафик ≥ 500 уникальных посетителей в день, иначе нет статистики.
- **Estimate:** 2–3 дня.

### 2.5 i18n — английская версия

- **Scope:** `en` локаль для B2B-клиентов из СНГ и экспорта.
- **Архитектура:** `lib/routes.ts` уже имеет поле `localizedPaths` — нужно только включить, добавить `next-intl` или `next-i18n-router`, перевести `lib/content/**` в en-версию.
- **Estimate:** 5–7 дней (включая перевод контента).

## Что было в v1 плане, но НЕ войдёт никогда

- **Lenis inertial scroll** — конфликтует с iOS momentum, ухудшает UX, не выживает Lighthouse audit.
- **Mouse parallax в hero** — killed на mobile, раздражает после запуска. Заменено на asymmetric CSS sticker-cluster.
- **`react-grid-layout`** dashboard widgets в админке — никто не перетаскивает виджеты. Static grid.
- **`@dnd-kit` kanban** — неделя edge cases (touch, scroll containers, a11y) за ноль пользовательской ценности. Dropdown статуса быстрее и честнее.
- **Custom icon set 1.75px weight** — 3 дня работы которые никто не заметит. Lucide purged под один вес достаточно.
