# Архитектура и Roadmap

## Текущее состояние

Лендинг из секций: Hero → Calculator → Catalog (5 продуктов) → Trust → CTA. Бэкенда, БД и оплаты пока нет.

## Слои (целевые)

```
app/          — роуты (App Router)
components/   — презентационные компоненты
lib/          — бизнес-логика, расчёты, типы
  ├── calculator.ts
  ├── db/           (будущее — Prisma клиент)
  ├── payments/     (будущее — ЮКасса)
  └── notifications/(будущее — Telegram)
```

Server Components по умолчанию. Client Components — только где нужна интерактивность (калькулятор, формы).

## Roadmap

- [ ] `/catalog/[slug]` — страницы продуктов
- [ ] `/order` — оформление заказа с загрузкой макета
- [ ] `/account` — личный кабинет
- [ ] `/account/track/[id]` — трекинг заказа
- [ ] Prisma + PostgreSQL (схема: User, Order, OrderItem, Product, Upload)
- [ ] Оплата СБП / ЮКасса
- [ ] `/admin` — панель управления
- [ ] Telegram-уведомления о заказах
- [ ] SEO-страницы и блог

## Принципы

- Единая дизайн-система через Tailwind-токены бренда (см. `CLAUDE.md`)
- Валидация на границах (формы, API-роуты) — zod или аналог
- Интеграции изолированы в `lib/` — компоненты не знают о ЮКассе/Telegram напрямую
- Строгая типизация публичных API модулей в `lib/`
